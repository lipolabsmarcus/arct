Custom Domains

Architect supports 3 types of urls for user sites. Regular (https://your-site.com/sites/some-name), sub-domain (https://some-name.your-site.com) and custom domain: (https://custom-domain.com). You can select which types are enabled from admin → settings → builder page.

Regular

Regular type does not require any extra setup, it will work out of the box. The regular url structure looks like this:

https://your-site.com/sites/1/site-name

Sub-domain

Sub-domain type will require you to setup what is called a wildcard sub-domain on your server / hosting. How exactly this is done depends on the hosting so you should refer to your hosting provider's documentation regarding this. You should make sure all sub-domains point to your main site:

https://*.your-site.com → https://your-site.com

Once this is done, when opening any sub-domain url on your site (for example https://books.your-site.com) architect will automatically resolve and show correct user site based on sub-domain name.

Custom Domains

Custom domains will allow users to attach their own custom domain (https://example.com) or sub-domain (https://site.example.com) to their account using A or CNAME records.

In order for this to work you will need to make sure that architect is the default directory for all requests to your server IP. How to do this depends on your hosting, so you will need to refer to hosting documentation.

For example, if you are on a VPS hosting that is using apache server, you can usually achieve this by modifying default apache conf file usually located in /etc/apache2/sites-enabled/ folder and named 000-default.conf. Open this file with text editor and replace default path /var/www/html/ in DocumentRoot and Directory with the path where architect is installed and restart Apache.

Note:
To check if custom domain is working, open it in the browser, you should either see architect homepage or the project this domain is attached to. If you get any other page, it means the server is handling the domain name and is pointing it to another page.
